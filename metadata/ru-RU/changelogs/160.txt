Falco 1.6.0

Релиз усиления безопасности. Два цикла аудита — открытых
находок Высокой или Средней серьёзности нет.

Новое:
- Опциональный режим «учётные данные с аппаратной привязкой»
  (Настройки → Безопасность). Повторная регистрация биометрии
  делает сохранённые токены недействительными.
- Подтверждение вводом + защита от tapjacking-атак во всех
  потоках удаления (Robot SSH, Cloud SSH, межсетевой экран, том,
  плавающий IP).
- Кнопка «Проверить учётные данные» на последнем шаге мастера.
- Предупреждение о буфере обмена для Android < 13.

Безопасность:
- TLS-пины обновлены: добавлен Let's Encrypt E8 для hel1/nbg1
  Object Storage. Предотвращает жёсткий лок при следующей ротации
  промежуточного сертификата.
- privateKey сертификатов Cloud + user_data cloud-init теперь
  маскируются в toString().
- Убран лишний биометрический запрос при первом выходе в
  передний план.

Изменено:
- androidx.security:security-crypto 1.1.0-alpha06 → 1.1.0 stable.
- androidx.biometric:biometric остаётся на 1.2.0-alpha05 (1.4.x
  требует AGP 8.9+ / compileSdk 36 — отложено до v2.0).
- ProGuard: -keep Hilt сужен до runtime SPI.
